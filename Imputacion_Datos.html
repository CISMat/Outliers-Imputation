IMPUTACI√ìN POR MODA

En el caso de la variable Mes, una imputaci√≥n muy sencilla ser√≠a tomar la moda para completar estos datos faltantes, sin embargo esta aplicaci√≥n puede generar errores.

# Moda

getmode <- function(v) {
  uniqv <- unique(v)
  uniqv[which.max(tabulate(match(v, uniqv)))]
}

Data_Impu_Moda<-ifelse(is.na(df$Mes),getmode(df$Mes),df$Mes)

El siguiente gr√°fico indica la imputaci√≥n de la variable Mes, la figura en color rojo son los datos conocidos y la figura en color verde es la imputaci√≥n, sin embargo se observa incosistencias en esta imputaci√≥n puesto que no se ajusta la imputaci√≥n con los datos de la base.

par(mfrow=c(1,1))
plot(density(df$Mes,na.rm = T),col=2,main="Mes") #ROJO CONOCIDO 
lines(density(Data_Impu_Moda),col=3) # VERDE IMPUTACION

IMPUTACI√ìN CON LA MEDIA

La librer√≠a mice permite realizar la imputacion de los datos mediante diferentes t√©cnicas. En particular con la funcion mice, se gener√°: Multivariate Imputations by Chained Equations (MICE).

Imputar con alg√∫n estad√≠stico de tendencia central (media, mediana, moda) consiste en cambiar todos los valores faltantes por el promedio de los datos que tenemos. Sin embargo el problema es que se esta disminuyendo artificialmente la variabilidad de la muestra.

# Media

imputed_data <- mice(df%>%select(Edad,Mes), method = "mean")
